# Workflow

## 1 日の流れ

1. 朝か作業前に日次ログを作る
2. その日の問いを 1 文で書く
3. 作業しながら `今日やったこと` と `参照・成果物` を追記する
4. 終了前に `得られたこと` と `スライド化メモ` を埋める
5. `次にやること` を 1 から 3 個だけ書く

補足:

- ログ本文は日本語でよい
- スライドとして外に出す内容は英語にする
- チャットで日本語の進捗を送ってくれれば、Codex がログと英語スライド素案へ整理する

## 週の流れ

1. 週末に日次ログを見返す
2. スライド素案を生成する
3. 週次レビューに、何が前進し何が詰まったかを書く
4. `status/next_actions.md` を更新する

## ログを書くときのルール

- 1 ログ 1 日 1 ファイル
- H1 / H2 / H3 を front matter に入れる
- H2 のログでは、扱ったデータセット名も本文に残す
- 実験・実装・読書・執筆を分けず、その日の研究としてまとめる
- ただし `スライド化メモ` は必ず箇条書きで残す

## スライド化メモに書くとよいこと

- 背景として使える一文
- その日の一番大きい前進
- 図にできる内容
- 結果として見せられる断片
- 次回スライドで言いたい主張

注意:

- このメモ自体は日本語でよい
- 実際のスライド原稿は英語に翻訳して `slides/outlines/` に置く
